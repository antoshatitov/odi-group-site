# Проверка structured data после деплоя

Дата инструкции: 05.06.2026

Эта инструкция нужна после деплоя изменений с JSON-LD/schema.org-разметкой на production.
Проверяем публичные страницы:

- `https://odi-group.ru/`
- `https://odi-group.ru/policy`
- `https://odi-group.ru/consent`
- `https://odi-group.ru/cookies`

## Что должно быть на сайте

На главной странице в DOM должен быть один невидимый `<script type="application/ld+json">` с
`@graph`, где есть:

- `Organization`
- `HomeAndConstructionBusiness`
- `WebSite`
- `WebPage`

На юридических страницах должен быть JSON-LD с:

- `WebPage`
- `BreadcrumbList`

Важно: отсутствие большого “rich result” в Google не всегда означает ошибку. Не все типы
Schema.org дают визуально расширенный сниппет. Главная цель проверки — убедиться, что разметка
читается поисковиками и не содержит критических ошибок.

## Google Rich Results Test

Инструмент: <https://search.google.com/test/rich-results>

1. Откройте Google Rich Results Test в браузере, где вы вошли в Google-аккаунт.
2. Оставьте вкладку `URL`.
3. Вставьте `https://odi-group.ru/`.
4. Нажмите `Проверить страницу`.
5. Дождитесь завершения проверки.
6. Проверьте результат:
   - не должно быть критических ошибок structured data;
   - если Google показывает “страница не поддерживает расширенные результаты”, но при этом нет
     ошибок парсинга, это может быть нормально для типов, которые не дают отдельный rich result;
   - если есть раздел с обнаруженными элементами, проверьте, что ошибки отсутствуют.
7. Повторите проверку для:
   - `https://odi-group.ru/policy`
   - `https://odi-group.ru/consent`
   - `https://odi-group.ru/cookies`

Что сохранить по результату:

- скриншот результата главной страницы;
- скриншот результата любой юридической страницы;
- список ошибок, если они появились.

## Яндекс Вебмастер: валидатор микроразметки

Инструмент: <https://webmaster.yandex.ru/tools/microtest/>

1. Откройте валидатор микроразметки Яндекс Вебмастера.
2. Войдите в Яндекс ID, если открылся экран авторизации.
3. Если Яндекс попросит выбрать сайт, выберите `odi-group.ru`.
4. Запустите проверку URL `https://odi-group.ru/`.
5. Проверьте, что валидатор видит JSON-LD/schema.org-разметку.
6. На главной странице ожидаем разметку организации/локального бизнеса и страницы сайта.
7. Повторите проверку для:
   - `https://odi-group.ru/policy`
   - `https://odi-group.ru/consent`
   - `https://odi-group.ru/cookies`
8. На юридических страницах ожидаем `BreadcrumbList` и `WebPage`.

Что считать успешным результатом:

- валидатор нашёл микроразметку;
- нет критических ошибок;
- предупреждения допустимы, если они относятся к необязательным полям и не требуют добавления
  невидимого/несуществующего на странице контента.

Что сохранить по результату:

- скриншот проверки главной страницы;
- скриншот одной юридической страницы;
- текст ошибок или предупреждений, если они появились.

## Дополнительная универсальная проверка Schema.org

Инструмент: <https://validator.schema.org/>

Эта проверка не заменяет Google/Yandex, но полезна как независимая проверка синтаксиса Schema.org.

1. Откройте Schema Markup Validator.
2. Нажмите `Run new test` / `Запустить новый тест`.
3. Выберите проверку по URL.
4. Вставьте `https://odi-group.ru/`.
5. Запустите проверку.
6. Убедитесь, что JSON-LD читается и нет синтаксических ошибок.
7. Повторите для юридических страниц, если Google или Яндекс показали спорный результат.

## Если валидатор показывает ошибки

1. Не добавляйте на сайт видимые FAQ, отзывы, рейтинги, цены или товары только ради валидатора.
2. Сначала проверьте, относится ли ошибка к обязательному или рекомендованному полю.
3. Если ошибка требует данные, которых нет на видимой странице, такую разметку лучше не добавлять.
4. Для текущего сайта нельзя размечать:
   - `FAQPage`, пока нет видимого FAQ-блока;
   - `Product`/`Offer`, пока нет реальных страниц домов с ценами/наличием;
   - `Review`/`AggregateRating`, пока нет видимых реальных отзывов.

## Контрольный итог

После проверки нужно зафиксировать:

- Google Rich Results Test: результат для главной и юридической страницы.
- Яндекс Валидатор микроразметки: результат для главной и юридической страницы.
- Были ли ошибки или только предупреждения.
- Нужны ли правки в JSON-LD.
